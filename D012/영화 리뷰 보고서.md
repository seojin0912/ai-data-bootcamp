# 씨네21 호프 리뷰 보고서 — 수집 리포트

## 1. 페이지 판별
- 정적/동적: **동적** (근거: 페이지 소스 보기(Ctrl+U)로 실제 리뷰 코멘트 문구를 검색했을 때 0건 검색됨 → 최초 HTML에는 리뷰 데이터가 없고, 브라우저에서 JS(`nzreview` jQuery 플러그인)가 실행된 후에야 화면에 리뷰가 렌더링됨. 렌더링 후 화면(및 Playwright로 받은 HTML)에서는 페이지당 5개씩 리뷰(`<li>`)가 확인됨)
- 선택한 도구: **Playwright + BeautifulSoup** (이유: 리뷰가 JS로 동적 로딩되는 구조라 `requests`만으로는 빈 틀만 받아지므로, 실제 브라우저를 실행해 렌더링을 완료시키는 Playwright가 필요했음. 렌더링된 HTML을 받은 뒤의 파싱은 BeautifulSoup으로 처리)

## 2. 수집 (AI 보조 4단계)
- 목적: 각 리뷰의 아이디·별점·코멘트(및 작성일자)를 표로 수집
- AI 초안 검증 결과: 행 수 **5개** → 문제: 한 페이지(1페이지)만 수집되고 있었음. 페이지네이션 버튼(`<a>2</a>`, `<a>3</a>`...)이 실제 링크가 아니라 JS 함수 호출(`nzreview('list', n)`) 방식이라, 단순히 첫 페이지 HTML만 파싱해서는 5개(1페이지 분량)에서 멈춤
- 수정: 버튼을 직접 클릭하는 대신 `page.evaluate("$('#netizen_review_area').nzreview('list', {page_num});")`로 JS 함수를 코드에서 직접 호출해 페이지 번호를 순회하도록 교정, 최종 **98행** 수집

## 3. 정제 (재적용한 랭글링)
- 결측치: 없음 (`id`, `date`, `rating`, `comment` 4개 컬럼 모두 98건 non-null 확인)
- 타입 변환: `rating`을 `pd.to_numeric`으로 숫자형(`int64`) 확인/변환
- 중복 제거: `df.duplicated().sum()` = 0 → 중복 없음 확인
- 파생 변수 생성: `comment_len`(코멘트 길이), `has_positive`/`has_negative`(긍/부정 키워드 포함 여부), `day`(날짜만 추출)

## 4. 윤리 검토
- robots.txt: `User-agent: SemrushBot`에 대해서만 `Disallow: /`가 명시되어 있고, 일반 크롤러(`User-agent: *`)에 대한 제한 규칙은 없어 **수집 가능**으로 판단
- Crawl-delay: 별도 명시 없음
- 요청 예의: 페이지 순회 시 `page.wait_for_timeout()`으로 각 요청 사이 대기시간을 두어 서버 부하를 줄임 (명시적 `time.sleep`/`User-Agent` 헤더 커스터마이징은 추가 적용 여지 있음)
- 저작권/개인정보: 아이디는 사이트 자체에서 이미 마스킹된 형태(`178*****`)로 제공되어 개인 식별 정보로 보기 어려움. 코멘트는 이용자의 창작물(저작권 있음)이므로 원문 그대로 재배포하지 않고 분석/통계 목적(길이, 키워드 포함 여부, 평점과의 관계 등)으로만 내부 활용

## 5. 다음 단계
- 분석 목적: **호불호가 얼마나 극단적으로 갈리는 영화인가?**
- 진행한 분석 및 결과:
  - **평점 분포/양극화**: 평균 5.59점, 표준편차 3.78로 매우 큼. 실제 분포(1점 30건, 10점 27건)가 양 극단에 몰려있고 중간(3~7점)은 상대적으로 적어 뚜렷한 **양극화(bimodal)** 확인
  - **코멘트 길이 vs 평점**: 8점(295자), 9점(155자) 그룹에서 코멘트가 특히 길게 나타남 — 강한 호평일수록 상세하게 서술하는 경향
  - **부정 키워드 vs 평점**: 부정 키워드(`최악`, `쓰레기`, `노잼` 등) 포함 리뷰의 평균 평점 3.50점, 미포함 리뷰 평균 5.83점으로 뚜렷한 차이 확인
  - **날짜별 추이**: 관찰 기간(7/17~7/24) 내내 표준편차가 3.2~4.4 수준으로 꾸준히 높게 유지 → 특정 시점 이후 갈린 게 아니라 **개봉 초반부터 지속적으로 호불호가 매우 심하게 갈리는 영화**로 해석 가능
- 추가로 분석하면 좋을 것: `has_positive`(긍정 키워드) 그룹의 평균 평점 결과도 함께 비교, 혹평(1~2점)/호평(9~10점) 그룹별 워드클라우드 비교로 "어떤 이유로 싫어하고 어떤 이유로 좋아하는지" 구체적 근거 보강
